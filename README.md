# Dev
## Run the dev server

For how to write articles (metadata, links, images, recipes, styles), see
[content/howto.md](content/howto.md).


Open the project in VS Code and press `Ctrl-Shift-B`. The default build task
starts the live-reload dev server and opens the Simple Browser to the drafts
page. Saving a file rebuilds the site and reloads the browser.

By hand:

```bash
./.venv/bin/python devserver.py
```

Or with make:

```bash
make livereload
```

The server runs on port `8109` and watches `content/**/*.md`,
`content/**/*.rst`, and the theme templates, CSS, and JS.

## Publishing

Publishing builds the site and pushes it live. Commit your work on the `write`
branch, then publish:

```bash
make github
```

`make github` runs Pelican against `publishconf.py`, commits the output to the
`master` branch, and pushes both `master` and `write`. GitHub Pages serves
`master`.

## Initial Setup

Create your virtual environment using `uv sync`.


# Architecture

## Git branches

- `write` — the branch you develop and commit on.
- `master` — the compiled output that GitHub Pages serves. Generated by
  `make github`; no manual commits.

## Hosting

Served as a GitHub Pages site from the `master` branch. The custom domain is
configured via the `content/extra/CNAME` file and the registrar's DNS records.
